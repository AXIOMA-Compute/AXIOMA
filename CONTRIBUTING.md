# Contributing to AXIOMA/Compute

Thank you for your interest in contributing.

This project welcomes contributions in:

- protocol design  
- cryptography  
- proof systems  
- constraint languages  
- JAM integration  
- research  
- documentation  

---

## Contribution Process

1. Fork the repository  
2. Create a feature branch  
3. Follow the style and structure of SPEC.md  
4. Submit a pull request with detailed explanation  

---

## Reporting Issues

Use GitHub Issues to report:

- specification inconsistencies  
- design flaws  
- security concerns  
- unclear documentation  
- proposed enhancements  

---

## Research Contributions

If contributing research:

- keep terminology consistent  
- cite cryptographic assumptions  
- provide formal definitions if possible  
- include diagrams or pseudocode  

---

## Code Style (future repositories)

- deterministic behavior only  
- no floating point usage  
- proofs must be reproducible  
- avoid undocumented constraints  

---

## Contact

For conceptual discussions, open a GitHub Discussion.
